# 6. Lessons Learned

1. Separate Implementation Acceptance from Repository Readiness.

Implementation quality and repository readiness are related but distinct engineering decisions. ESR-0006 benefited from making that separation explicit.

2. Runtime artefacts are not governance artefacts.

Prototype JARVIS chat exports were classified as future product runtime artefacts rather than AIEMS controlled artefacts or repository defects.

3. OSE provides measurable engineering value.

Verified WikiLinks, related artefact sections and graph navigation improved traceability and repository comprehension without changing controlled engineering meaning.

4. Obsidian provides significant engineering navigation benefits.

Obsidian improved local navigation, relationship discovery and knowledge visualisation while remaining compatible with repository-first Markdown governance.

5. AIEMS remains AI-agnostic.

OSE and Obsidian support AIEMS, but AIEMS remains independent of a specific tool, model, vendor or workspace.

6. Evidence precedes standardisation.

ESR-0006 working practices were validated through use. They remain working practices until future review determines whether any should become formal standards.
