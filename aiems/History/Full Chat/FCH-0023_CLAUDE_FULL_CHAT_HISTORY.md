# 5. Source Preservation Notes

Like FCH-0015 through FCH-0020, this artefact was generated by Claude parsing its own Claude Code session log (JSONL format) directly.

Extraction method: the session log was read in full (2,024 raw log lines, from session opening to this artefact's creation). Only genuine conversational text content was extracted - Programme Sponsor messages (`origin.kind == "human"`) and Claude's narrative text responses. Tool invocations, tool results, internal reasoning ("thinking") blocks, and session-management entries (queue operations, attachments, file-history snapshots, auto-generated titles, last-prompt tracking) were excluded, as they are execution mechanics rather than conversational content. One skill-invocation boilerplate block (the `run` skill's own instructions, injected as a synthetic non-human turn) was excluded as mechanical rather than conversational; one synthetic `[Request interrupted by user]` marker was retained, since it records a genuine conversational event (the Programme Sponsor interrupting mid-turn) rather than execution mechanics. Consecutive same-speaker text segments (interleaved with tool calls, on the Claude side) were merged into a single turn. Content is reproduced verbatim from the source log in original order; normalisation was limited to Markdown section formatting, not reinterpretation.

Extraction was performed programmatically (a Python script parsing the JSONL line by line, classifying each entry and merging consecutive same-speaker text segments), matching the FCH-0015 through FCH-0020 precedent, rather than manually retyped from memory, to guarantee verbatim fidelity across a very long, multi-Work-Package session.

**Coverage note**: this artefact was generated mid-session-report-of-itself - that is, while Claude was in the process of writing this very artefact, the session log continued to accumulate. The extraction captures the session faithfully through the point the extraction script was run; the final few turns discussing the extraction's own completion are necessarily not included, the same structural limitation FCH-0020 and its predecessors also carry.
